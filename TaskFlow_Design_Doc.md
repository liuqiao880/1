## 1. 项目概述

- **应用名称**：TaskFlow
- **核心定位**：离线优先，AI 辅助的轻量级个人任务管理工具。
- **设计哲学**：极简、高效、可控。界面清爽无冗余元素，交互反馈细腻，所有 AI 行为完全由用户手动触发，尊重用户掌控感。

---

## 2. 核心功能清单

### 2.1 任务层级管理

- 支持"父任务"与"子任务"结构。
- 用户输入父任务（总目标），AI 拆解出子任务（具体执行步骤）。
- 父任务右侧以横向细进度条展示子任务完成度。

### 2.2 AI 智能规划（核心亮点）

- **手动触发**：仅在用户点击「✨ AI 规划」按钮时调用 API。
- **自动拆解**：根据任务标题、描述和截止日期，生成带时间节点的子任务清单。
- **动态重排**：用户汇报进度后，手动触发 AI 重新调整后续日期任务。
- **预览确认**：AI 生成/重排结果先预览，用户确认后才保存，支持编辑调整。
- **失败降级**：AI 调用失败时，保留用户输入内容，提示"AI 暂时不可用，已保存为普通任务"，并提供重试按钮。

### 2.3 任务基础操作（CRUD）

- 新增、删除、修改任务。
- 点击复选框一键切换"完成/未完成"状态（带触感反馈与删除线动画）。
- 左滑删除，右滑标记完成（手势操作）。
- 删除后 3 秒内可通过 Snackbar 撤销。
- 长按进入多选模式，支持批量删除/完成/移动日期。

### 2.4 动态时间线视图

- 任务按"今天"、"明天"、"本周"、"更晚"自动分组。
- AI 调整任务日期后，任务卡片自动"漂移"到对应分组。
- 顶部快速筛选标签，一键切换时间线视图。

### 2.5 搜索功能

- 顶部导航栏搜索入口，支持标题 + 描述模糊搜索。
- 搜索结果高亮关键词。
- 空结果友好提示。

### 2.6 到期系统通知

- 使用 WorkManager 每日早晨推送通知提醒今日待办。
- 不写入系统日历，保持轻量。

## 3. UI/UX 交互优化方案

### 3.1 视觉美观优化

| 优化项 | 具体实现 |
|--------|----------|
| 色彩层次 | 主色 #2e7d32（墨绿）；优先级色条：紧急红(#d32f2f)、普通黄(#f9a825)、低优先级蓝(#42a5f5)；已完成任务：灰色文字 + 删除线 + 降低透明度 |
| 统一样式规范 | 使用 CSS 变量统一管理圆角（16px）、阴影（极淡卡片阴影）、间距；任务卡片背景纯白，外壳大阴影分层 |
| 可视化进度条 | 父任务右侧以横向细进度条（宽 60px）+ 百分比文字展示子任务完成度 |
| 精致间距 | 分组间距 18px，子任务内边距均衡，视觉紧凑透气 |
| 暗色模式 | 基于 CSS 变量一键切换暗色主题，V1 同步交付 |
| 空状态设计 | 首次启动/筛选无结果/搜索无结果时，配简约插画 + 友好引导文案 |

### 3.2 交互体验优化

| 优化项 | 具体实现 |
|--------|----------|
| 全局状态记忆 | 折叠/展开状态、复选框完成状态、筛选偏好自动存入 DataStore，刷新页面或下次打开时恢复 |
| 触感与动效 | 点击复选框触发设备震动（navigator.vibrate / HapticFeedback）；折叠展开采用弹性缓动曲线（cubic-bezier(0.34, 1.56, 0.64, 1)）；按钮点击带有轻微缩放下沉反馈 |
| 父任务折叠/展开 | 点击父任务整行，右侧箭头平滑旋转 90°，子任务列表以手风琴形式展开或收回 |
| 任务勾选完成 | 点击任务前复选框，文字即时添加/移除删除线，伴随透明度变化，反馈清晰无延迟 |
| AI 规划加载状态 | 点击 AI 按钮后，按钮变为进度环并禁用；主列表区域显示流光骨架屏（Shimmer 效果） |
| 手势操作 | 左滑露出删除按钮，右滑标记完成；长按任务进入多选批量操作模式 |
| 撤销操作 | 删除任务后底部弹出 Snackbar，3 秒内可撤销，防误删且不打断流程 |
| 相对时间显示 | 任务卡片显示"2 分钟前创建"等相对时间，比绝对时间戳更友好 |

### 3.3 移动端 & 平板适配优化

| 优化项 | 具体实现 |
|--------|----------|
| 滚动体验 | 启用流畅滚动；自定义滚动条为 4px 细浅灰色，柔和融入背景 |
| 点击热区 | 所有可交互元素最小高度/宽度不低于 44dp，确保手指或触控笔轻松点击 |
| 响应式布局 | 手机竖屏自适应；平板（768px+）自适应优化布局；横屏模式自动适配，防止拉伸变形 |
| 视觉防误触 | 屏蔽双击缩放，禁止文字被选中拖拽 |

### 3.4 代码健壮性 & 可维护性

| 优化项 | 具体实现 |
|--------|----------|
| CSS 变量 / Material Theme | 颜色、圆角、间距统一抽取，一键切换主题（含暗色模式） |
| 事件委托架构 | 使用状态驱动 + 事件回调，避免大量行内点击耦合，逻辑集中管理 |
| 命名规范 | 采用清晰命名风格（如 TaskItem、ParentTaskArrow），避免样式冲突 |
| 数据持久化封装 | 状态存储封装独立函数，底层 DataStore / Room 可平滑迁移 |
| 数据库迁移预留 | Room 提前配置 Migration 机制，V1 预留备用字段降低后期改表成本 |
| 崩溃采集 | 集成 Bugly / Sentry，线上问题可追溯 |

### 3.5 功能布局说明

**已明确移除底部固定操作栏**，保持主界面视觉纯净：

- **新增任务** → 右下角悬浮 FAB 按钮（"+"）
- **筛选视图** → 顶部标签栏承担
- **同步操作** → 顶部菜单或设置页面（后期迭代）
- **搜索** → 顶部导航栏搜索图标
- **AI 规划** → FAB 长按展开 / 任务详情页入口

## 4. 技术栈选型

| 层级 | 技术选型 |
|------|----------|
| 开发语言 | Kotlin（100%） |
| UI 框架 | Jetpack Compose（声明式 UI） |
| 架构模式 | MVVM + UseCase + Repository（单向数据流） |
| 本地数据库 | Room（配合 Kotlin Flow 自动刷新） |
| 本地存储 | DataStore（存储 API Key、折叠状态、偏好设置等） |
| 网络请求 | Retrofit + OkHttp（调用通用 AI 接口） |
| 异步处理 | Kotlin Coroutines + Flow |
| 依赖注入 | Hilt（推荐，便于后期解耦） |
| 图片/动效 | Coil（加载图标）、Accompanist（Shimmer 骨架屏、权限等） |
| 崩溃采集 | Bugly / Sentry（二选一） |

### 架构分层示意

```
┌─────────────────────────────────┐
│  UI 层 (Compose + ViewModel)    │
├─────────────────────────────────┤
│  UseCase 层 (业务用例)           │  ← 新增：每个用例独立类，便于单测
├─────────────────────────────────┤
│  Repository 层 (数据仓库)        │
├─────────────────────────────────┤
│  Data 层 (Room / DataStore)     │
│  Network 层 (Retrofit)          │
└─────────────────────────────────┘
```

## 5. 数据结构（Room 实体）

### 5.1 Task 实体

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | Int (主键) | 唯一标识，自增 |
| title | String | 任务标题（非空，1-50 字符） |
| description | String? | 描述（可空，最大 200 字符） |
| status | TaskStatus | 任务状态枚举（TODO / COMPLETED，预留 IN_PROGRESS / CANCELLED） |
| priority | Int | 1 高 / 2 中 / 3 低（默认 2） |
| dueDate | Long? | 截止日期时间戳（可空） |
| parentId | Int? | 父任务 ID（null 表示顶级任务） |
| order | Int | 同级任务排序权重（默认 0，数字越小越靠前） |
| aiGenerated | Boolean | 是否由 AI 生成（默认 false） |
| syncStatus | SyncStatus | 同步状态（LOCAL_ONLY / SYNCED / PENDING，V1 全为 LOCAL_ONLY） |
| createTime | Long | 创建时间戳（自动生成） |
| updateTime | Long | 更新时间戳（自动更新） |
| extra1 | String? | 备用字段 1（预留，降低后期迁移成本） |
| extra2 | Int? | 备用字段 2（预留） |

### 5.2 枚举定义

```kotlin
enum class TaskStatus {
    TODO,        // 待办
    COMPLETED,   // 已完成
    IN_PROGRESS, // 进行中（V2 启用）
    CANCELLED    // 已取消（V2 启用）
}

enum class SyncStatus {
    LOCAL_ONLY,  // 仅本地
    SYNCED,      // 已同步
    PENDING      // 待同步
}
```

### 5.3 父子任务删除策略

- 删除父任务时弹出确认，提示"将同时删除 N 个子任务"
- 使用 Room 事务（Transaction）执行级联删除，保证数据一致性
- 删除后 3 秒内可通过 Snackbar 撤销（事务 + 软标记实现）

## 6. AI 模块设计

### 6.1 AI Provider 抽象层

为支持多种 AI 模型（OpenAI 格式、Gemini、Ollama 等），定义统一接口：

```kotlin
interface AiProvider {
    suspend fun generateSubtasks(
        parentTask: Task,
        config: AiConfig
    ): Result<List<Task>>

    suspend fun rescheduleTasks(
        tasks: List<Task>,
        progressReport: String,
        config: AiConfig
    ): Result<List<Task>>
}
```

不同 Provider（OpenAIProvider、GeminiProvider 等）各自实现，配置页选择 Provider 类型。

### 6.2 AI 配置项

设置页可配置：

| 配置项 | 说明 |
|--------|------|
| Provider 类型 | OpenAI 兼容 / Gemini / 其他 |
| Base URL | API 地址 |
| API Key | 用户自己的密钥（本地加密存储） |
| Model | 模型名称 |
| 超时时间 | 请求超时（默认 30 秒） |
| System Prompt（高级） | 自定义 AI 规划的系统提示词（高级用户可调整风格） |

### 6.3 AI 交互流程

**生成子任务流程：**
1. 用户点击「✨ AI 规划」→ 弹出输入框（目标 / 补充需求）
2. 点击生成 → 按钮变进度环，列表区显示 Shimmer 骨架屏
3. API 返回 → 展示**预览列表**，用户可：
   - 勾选/取消子任务
   - 编辑标题、日期、优先级
   - 拖拽调整顺序
4. 点击「确认保存」→ 写入数据库，父任务展开显示

**重排任务流程：**
1. 用户在父任务页点击「🔄 AI 重排」→ 输入当前进度
2. AI 返回调整建议 → 展示**新旧对比**（日期变化高亮）
3. 用户确认 → 应用调整，任务卡片动画漂移到对应分组
4. 提供「撤销重排」入口

### 6.4 使用统计

记录 AI 调用情况（高级设置页可查看）：
- 本周/本月调用次数
- 成功 / 失败次数统计
- 如 API 返回 token 用量则同步记录
